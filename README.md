# Effective Mobile — Система аутентификации, авторизации и записи пользователей.

## Описание проекта

Backend-приложение на Django с реализованной собственной системой аутентификации, авторизации и разграничения прав
доступа.

Проект включает:

* регистрацию пользователей;
* вход и выход из системы;
* редактирование профиля;
* soft delete пользователей;
* разграничение прав доступа;
* административную панель управления;
* систему записи пользователей в офисы.

---

# Функционал

## Аутентификация

### Регистрация

Пользователь может зарегистрироваться, указав:

* username
* email
* имя - опционально
* фамилию - опционально
* отчество - опционально
* пароль

---

### Login

Вход осуществляется по почте и паролю.

После успешной аутентификации Django Session Middleware идентифицирует пользователя при последующих запросах.

---

### Logout

Пользователь может выйти из системы.

---

### Soft Delete

Удаление пользователя реализовано через поле:

```python
is_active = False
```

После деактивации пользователь не может войти в систему, однако запись сохраняется в базе данных.

# Система ролей

В проекте реализовано разграничение доступа на основе ролей.

## Роли

| Роль  | Описание             |
|-------|----------------------|
| USER  | Обычный пользователь |
| STAFF | Сотрудник            |
| ADMIN | Администратор        |

---

# Разграничение доступа

## USER

Имеет доступ к:

* просмотру профиля;
* редактированию профиля;
* просмотру записей;
* созданию записей;
* просмотру собственных записей.

---

## STAFF

Дополнительно имеет доступ к:

* панели сотрудников;
* редактированию расписания;
* управлению слотами записи.

---

## ADMIN

Имеет полный доступ к системе:

* управление пользователями;
* управление сотрудниками;
* управление офисами;
* редактирование данных;
* soft delete пользователей.

---

# Система проверки доступа

В проекте реализованы собственные decorators:

```python
@authenticate_required
@staff_required
@superuser_required
```

---

## Логика работы

### 401 Unauthorized

Если пользователь не аутентифицирован:

```python
return HttpResponse(status=401)
```

---

### 403 Forbidden

Если пользователь аутентифицирован, но не имеет доступа:

```python
return HttpResponseForbidden()
```
